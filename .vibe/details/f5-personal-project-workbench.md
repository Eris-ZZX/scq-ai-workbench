# F5 个人项目工作台 — Build Ready

## 1. 目标

F5 将原“个人工作台”和“项目工作台首页”合并为一个统一入口：个人项目工作台。

页面要回答两个问题：

- 我现在先处理什么？
- 我进入某个项目后，如何把任务推进掉？

具体处理不再发生在孤立抽屉中，而是进入项目工作区，在项目上下文中处理具体任务。

## 2. 信息架构

### 统一入口

- 侧边栏只保留一个业务主入口：个人项目工作台。
- 不再并列展示“个人工作台”和“项目工作台”两个入口。
- 首页上半区是我的待处理任务，下半区是我的项目。

### 项目工作区

- 点击待办：进入项目工作区，定位到对应任务并展开详情。
- 点击项目：进入项目工作区，默认展开当前阶段。
- 项目工作区是具体业务操作发生的地方。

## 3. 首页规格

### 我的待处理任务

- 默认展示统一优先级列表。
- 不按项目/任务类型分组作为默认展示。
- 支持轻量筛选：全部、逾期、阻塞、退回、待确认关闭。
- 点击任务后跳转到项目工作区并展开该任务详情。

### 我的项目

采用推进型项目卡片。卡片展示：

- 项目名
- 当前阶段
- 整体进度
- 下一待处理
- 阻塞/逾期/待确认关闭标记

项目区支持按项目名搜索。

下一待处理按项目推进优先级选取：

1. 阶段门卡住
2. 阻塞
3. 逾期
4. 待确认关闭
5. 我的责任项

## 4. 项目工作区规格

### 顶部摘要条

轻量展示项目上下文：

- 项目名
- 当前阶段
- 进度
- 逾期/阻塞/待确认关闭数量
- 阶段门状态

### 左侧结构区

- 展示阶段 / 项目活动 / 子任务三层结构。
- 当前阶段默认展开。
- 其他阶段默认折叠，可展开。
- 每层显示状态标签和数量徽标。
- 阶段显示异常数量。
- 项目活动显示完成、阻塞、逾期、待确认关闭等状态。
- 子任务显示自身状态。

### 右侧详情面板

详情面板采用上信息、中提交、下历史结构。

上：任务信息

- 阶段
- 项目活动
- 子任务
- 责任角色
- 计划时间
- 当前状态

中：提交与动作

- 提交标准
- 附件
- 完成说明
- 阻塞说明
- 不涉及原因
- 角色化动作按钮

下：历史

- 当前任务历史
- 所属项目活动关键历史
- 包含完成、退回、代维护、上传附件、确认关闭、不涉及等关键留痕。

## 5. 权限与动作

主规则：角色决定能否操作。

副规则：任务类型决定具体操作。

- 当前责任角色：可维护自己的责任子任务，包含完成、上传附件、填写说明、标记阻塞。
- NPQ：可查看和维护项目内任务，可代维护、退回、标记不涉及、确认关闭项目活动。
- 管理者：只读，查看项目状态、风险、历史。
- 管理员：不参与业务处理，业务区默认只读或不展示处理动作。

所有 NPQ 代维护、退回、关闭、不涉及操作必须留痕。

## 6. 第一版不做

- 审批流
- 交付件在线预览
- 外部通知
- 复杂报表筛选
- 保存筛选视图
- 完整风险管理模块
- 全项目流水账式历史

## 7. 验收标准

- 用户登录后业务入口显示为“个人项目工作台”。
- 首页上方展示按优先级排序的我的待处理任务。
- 首页待办支持全部、逾期、阻塞、退回、待确认关闭筛选。
- 首页下方展示我的项目，并支持项目名搜索。
- 项目卡能显示下一待处理，并按项目推进优先级计算。
- 点击待办后进入对应项目工作区，自动选中并展开该任务详情。
- 点击项目后进入项目工作区，默认展开当前阶段。
- 项目工作区顶部显示轻量项目摘要条。
- 左侧结构默认当前阶段展开，其他阶段折叠。
- 左侧阶段/项目活动/子任务每层都有状态标签和数量徽标。
- 右侧详情按上信息、中提交、下历史组织。
- NPQ 和当前责任角色能看到可执行动作。
- 管理者只读，管理员不参与业务处理。
- 当前任务历史和所属项目活动关键历史可见。
