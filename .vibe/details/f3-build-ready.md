# F3 Build-Ready 参考

> 用途：F3 阶段实现参考，保留当时的模块设计、边界和验收口径。它不是正式 PRD & SPEC；正式 PRD & SPEC 只看 `00-docs/strategy/scq-ai-strategy-technical-plan.md`。

## 1. 定位

F3 = **NPQ 活动流真实运转 + 配套后台底座完善**。

F3 不横向扩更多业务线，而是把 F2 的活动清单升级为真实项目管理系统：可配置、可分派、可追踪、可复盘。

## 2. 核心原则

- 聚焦 NPQ 活动流。
- 后台底座只服务当前业务流，不做全平台通用引擎。
- 模板由系统内结构化维护，F3 不做 Excel 导入/导出。
- 项目创建后使用项目活动快照，后台模板更新默认不影响已创建项目。
- 一线执行者通过“我的待办”工作，不靠翻项目表格找任务。
- 所有关键变更留动态记录或运行日志。

## 3. 用户与岗位

- admin：维护模板中心、岗位、用户岗位、权限矩阵、运行日志。
- NPQ：创建项目、任命岗位负责人、调整项目快照、批量操作、退回子任务、关闭母任务、阶段门通过/条件放行。
- 项目岗位负责人：处理自己的子任务、上传附件/填写说明、标记阻塞、完成任务、处理退回。
- 管理层/授权角色：查看项目进度、阶段状态、到期/逾期、角色按时完成率。

岗位规则：

- 一个用户只能绑定一个岗位。
- 一个用户可被任命到多个项目。
- 模板子任务挂责任岗位。
- 项目创建时按项目岗位任命映射具体负责人。
- 岗位未任命时，项目创建向导必须提示缺口。

## 4. 模板中心

支持多个内置模板。每个模板只把最新已发布版本提供给新项目使用。

模板能力：

- 模板列表
- 新建模板
- 复制模板
- 编辑草稿
- 发布
- 停用
- 三级结构维护

三级结构：

- 阶段
- 母任务
- 子任务

子任务模板字段：

- 子任务名称
- 责任岗位
- 交付件名称
- 是否需要附件
- 是否需要完成说明
- 默认计划偏移
- 排序
- 启停

不做：

- Excel 导入/导出
- 模板发布审批
- 条件分支/前置依赖/规则表达式
- 前台选择历史版本

## 5. 项目创建向导

轻量 4 步：

1. 项目信息
2. 选择内置模板，系统使用最新已发布版本
3. 任命项目岗位负责人
4. 预览并生成项目活动快照

规则：

- 未发布模板不可选。
- 岗位负责人缺口必须提示。
- 生成快照后，后台模板更新不影响该项目。

## 6. 项目活动快照

NPQ 可有限调整项目快照：

- 新增项目内母任务/子任务
- 修改责任岗位/负责人
- 修改计划时间
- 修改交付件要求
- 修改排序
- 标记不涉及
- 恢复涉及

不建议物理删除任务。

“不涉及”规则：

- 不进入完成率
- 不进入母任务关闭条件
- 不进入逾期
- 不进入我的待办
- 不进入角色按时完成率分母
- 不进入风险/阻塞统计
- 只作为项目活动结构中的说明性记录展示

## 7. 子任务状态

F3 子任务状态：

- 未开始
- 进行中
- 已退回
- 已完成

移除 F2 的“已提交”。

完成校验：

- 需要附件时，至少有一个有效附件。
- 需要说明时，完成说明必填。
- 不涉及任务不允许完成动作。

NPQ 退回：

- NPQ 可退回进行中或已完成子任务。
- 必须填写退回原因。
- 状态变为已退回。
- 进入负责人待办。
- 写入动态记录。

## 8. 附件

轻量附件上传：

- 子任务可上传多个附件。
- 记录文件名、大小、类型、上传人、上传时间。
- 支持下载。
- 支持删除，删除必须留操作日志/动态记录。

不做：

- 在线预览
- 文件版本
- 全文检索
- 完整文件中心

## 9. 我的待办与站内提醒

我的待办分组：

- 待处理
- 已退回
- 即将到期
- 已逾期
- 待补交付件
- 我负责的待关闭母任务

站内提醒触发：

- 任务被分派
- 任务被退回
- 即将到期
- 已逾期
- 母任务待关闭

F3 暂不接钉钉/邮件/企业微信。

## 10. 批量操作

NPQ/admin 可用：

- 批量修改计划时间
- 批量修改负责人
- 批量修改责任岗位
- 批量标记不涉及
- 批量恢复涉及
- 批量退回

所有批量操作必须留痕。

## 11. 权限矩阵

F3 做 NPQ 活动流动作级权限矩阵，不做全平台通用权限引擎。

动作包括：

- 查看项目
- 创建项目
- 编辑项目
- 维护模板
- 任命项目岗位负责人
- 上传/删除附件
- 批量操作
- 标记不涉及
- 退回子任务
- 关闭母任务
- 阶段通过/条件放行
- 查看运行日志

admin 必须保留兜底权限，避免系统自锁。

## 12. TR 阶段门

阶段：

- TR1
- TR2&3
- TR4
- TR4A
- TR5
- TR6

规则：

- 项目有当前阶段。
- 当前阶段聚焦展示。
- 阶段通过后自动进入下一阶段。
- 暂不做复杂审批链和阶段评审表单。

阶段切换前置条件：

- 当前阶段涉及母任务原则上关闭。
- 当前阶段无阻塞任务。
- 存在逾期、未完成例外或其他风险时，NPQ 必须填写条件放行说明并留痕。

## 13. 时间视图

F3 做轻量时间视图，不做甘特图。

包括：

- 阶段时间线
- 当前阶段剩余事项
- 即将到期任务列表
- 逾期任务列表

## 14. 运行日志与动态记录

后台菜单使用“运行日志”，替代“可观测性”面向用户展示。

运行日志 tabs：

- 操作日志
- 系统错误
- 接口请求

业务详情中使用“动态记录”：

- 项目动态
- 母任务动态
- 子任务动态
- 附件上传/删除
- 批量操作
- 阶段门通过/条件放行

## 15. 建议数据模型

F3 关键新增/调整对象：

- ActivityTemplateSet
- ActivityTemplateVersion
- ActivityTemplateStage
- ActivityTemplateParent
- ActivityTemplateChild
- PositionRole
- UserPosition
- ProjectPositionAssignment
- ProjectActivitySnapshotMeta
- ActivityAttachment
- Notification
- NpqActionPermission
- StageGateRecord

F2 对象需要调整：

- ProjectActivityChild 增加 `assigneeUserId`、`responsibleRoleId`、`isNotApplicable`。
- 子任务状态移除 `submitted`，增加/使用 `returned`。
- ActivityEvent 继续作为动态记录基础。
- Observability 后台展示名改为运行日志。

## 16. F3 不做

- Excel 导入/导出
- 正式审批流
- 交付件在线预览
- 外部通知
- 独立项目风险清单
- 甘特图
- 全平台通用权限引擎
- 模板发布审批
- 完整文件中心
- AI 功能扩展

## 17. 验收标准

- 模板中心可维护多模板和三级结构。
- 新项目只使用模板最新已发布版本。
- 项目创建向导可完成模板选择、岗位任命和快照生成。
- 用户单岗位、多项目任命生效。
- 我的待办能显示当前用户责任项。
- 子任务附件上传/下载/删除可用且留痕。
- 子任务状态为未开始/进行中/已退回/已完成。
- NPQ 可退回子任务并进入负责人待办。
- 不涉及任务不进入任何计算。
- 轻量批量操作可用且留痕。
- 站内提醒和待办角标可用。
- 动作级权限矩阵约束关键动作。
- TR 阶段门和条件放行可用。
- 阶段时间线、到期/逾期列表可用。
- 后台运行日志重命名和分组完成。
- lint、typecheck、unit/integration test、build、本地冒烟通过。

## 18. Ready-to-use build prompt

基于 `.vibe/plan.md` 和本文件实现 F3。重点把 F2 活动清单升级为真实 NPQ 项目管理系统：模板中心、项目向导、岗位任命、项目快照、我的待办、附件、退回、不涉及、批量操作、站内提醒、权限矩阵、TR 阶段门、运行日志和时间视图。不得实现 F3 明确排除项。完成后运行 lint、typecheck、unit/integration test、build、本地冒烟，并同步 Vibe Tracker。
