# 质量工作台 - 常驻计划

> 用途：这是 Vibe Tracker 的日常控制面板。它只保留项目大背景、当前判断规则、未完成/待决事项和参考入口；正式 PRD & SPEC 只看 `00-docs/strategy/scq-ai-strategy-technical-plan.md`；已经完成的功能细节放到 `.vibe/details/completed-features-reference.md`。

## 1. 项目大背景

质量工作台是面向质量工程师日常工作的整合型工作台，不是单一的新产品活动系统。

核心目标：

- 把质量工程师每天要看的项目、待办、任务、风险、证据、配置和运行记录收敛到一个工作台入口。
- 让用户登录后先看到“我现在要处理什么、哪个项目最需要推进、下一步动作是什么”。
- 以项目推进和个人待处理事项为主线，减少散落页面、重复入口和只适合演示的功能堆叠。
- 新产品活动 / NPQ 只是当前已经落地的一个业务模块，用来验证工作台的任务、项目、权限、证据和留痕能力。

产品边界：

- 工作台主线优先：统一入口、个人待办、项目工作区、配置中心、运行日志、跨模块导航与协作体验。
- 通用工作流能力其次：任务分派、状态流转、附件证据、动态记录、角色化动作、提醒、项目摘要。
- NPQ / 新产品活动模块保持小而专注：只承载新产品导入活动跟踪和 NPQ 活动流，不反向定义整个系统。

## 3. 后续规划原则

- 先维护工作台体验，再扩展业务模块。
- 新功能必须说明它服务的是“工作台主线”“通用工作流”还是“某个业务模块”。
- 已完成模块默认不继续扩张，除非它暴露出影响工作台主线的真实问题。
- 避免把 NPQ 的术语、阶段、流程和权限设计硬塞进其他质量场景。
- 新业务线进入前，优先抽象可复用能力：待办、项目空间、证据、状态、角色动作、通知和报表。
- 审批、在线预览、外部通知、完整文件中心、复杂报表等仍作为后续扩展，不抢占当前主线。
