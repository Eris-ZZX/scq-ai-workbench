## 3. F3 核心决策摘要

- F3 聚焦 **NPQ 活动流真实运转 + 配套后台底座完善**。
- 不横向扩 SQE/PQE/知识库/AI 等新业务线。
- 后台底座只服务 NPQ 活动流，不做全平台通用引擎。
- 模板中心支持多个内置模板；新项目只使用某模板的最新已发布版本。
- 模板用系统内结构化三级编辑维护，F3 不做 Excel 导入/导出。
- 项目创建后生成项目活动快照；后台模板更新默认不影响已创建项目。
- 用户只能绑定一个岗位，但可被任命到多个项目。
- 项目内按岗位任命负责人，模板子任务责任岗位映射到具体负责人。
- F3 做轻量“我的待办/我的责任项”。
- 子任务状态：未开始、进行中、已退回、已完成；移除已提交。
- NPQ 可退回子任务，必须填写退回原因并留痕。
- 交付件做轻量附件上传，不做在线预览。
- NPQ 可有限调整项目活动快照；用“不涉及”标记项目例外。
- 不涉及任务不进入任何计算，只保留说明性展示。
- NPQ/admin 支持轻量批量操作，所有操作留痕。
- 通知先做站内提醒，外部通知放 F5。
- 权限做 NPQ 活动流动作级权限矩阵，不做全平台权限引擎。
- 后台“可观测性”面向用户改名为“运行日志”；业务详情叫“动态记录”。
- 项目引入简化 TR 阶段门：TR1、TR2&3、TR4、TR4A、TR5、TR6。
- 阶段切换规则：关闭 + 无阻塞 + 条件放行说明。
- 时间视图做轻量阶段时间线 + 到期/逾期列表，不做甘特图。

## 6. F4 工作台可用性重构

详见 `.vibe/details/f4-workbench-alignment.md`。

核心决策：

- 新建 `/workbench` 业务主入口。
- 导航收敛为工作台 + 后台配置。
- 单页行动优先工作台，不做二级视图切换。
- 默认展示当前用户参与/任命相关的未完成项目。
- 待办按项目分组，项目内按风险/时间排序。
- 项目卡轻量展示，并按关注程度排序。
- 行动指标为待处理、逾期、阻塞、缺交付件、待关闭。
- 待办点击后进入右侧处理抽屉，动作按角色显示。
- 项目详情页改为项目待办优先，下方保留活动结构。
- 不新增数据表，不做搜索，不做星标项目。
- F4 验收采用角色优先，seed 固定 8 个测试账号：`npq/pqe/sqe/fae/ram/qcm/manager/admin`。

## 7. F5 个人项目工作台

详见 `.vibe/details/f5-personal-project-workbench.md`。

核心决策：

- 合并原个人工作台和项目工作台首页，统一入口命名为“个人项目工作台”。
- 首页上半区为“我的待处理任务”，下半区为“我的项目”。
- 我的待处理任务采用统一优先级列表，筛选保留全部、逾期、阻塞、退回、待确认关闭。
- 我的项目采用推进型项目卡片：项目名、当前阶段、进度、下一待处理、阻塞/逾期/待确认关闭标记。
- 下一待处理按项目推进优先级选取：阶段门卡住 > 阻塞 > 逾期 > 待确认关闭 > 我的责任项。
- 点击待办进入项目工作区的具体任务并展开详情；点击项目进入项目工作区并默认展开当前阶段。
- 项目工作区顶部为轻量项目状态摘要条。
- 项目工作区左侧为阶段/项目活动/子任务结构，当前阶段展开，其他阶段折叠。
- 左侧每层显示状态标签和数量徽标。
- 右侧任务详情面板采用上信息、中提交与动作、下历史结构。
- 历史区显示当前任务历史 + 所属项目活动关键历史。
- 动作按角色展示；NPQ + 当前责任角色可编辑，管理者只读，管理员不参与业务处理。
- F5 不做审批、交付件在线预览、外部通知、复杂报表筛选和完整风险管理。

实施 Backlog：

- [x] F5.S2 重构首页：我的待处理任务统一优先级列表、轻量筛选、我的项目搜索和推进型项目卡片。
- [x] F5.S3 实现项目工作区基础布局：顶部项目摘要条、左侧阶段/项目活动/子任务结构、右侧详情面板。
