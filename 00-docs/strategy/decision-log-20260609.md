### 目录结构

```text
quality-workbench-app/src/
├─ platform/          ← 共享平台能力（将来可抽为 @qe/platform）
│  ├─ auth/           ← 登录、会话、UserContext
│  ├─ permissions/    ← 项目+角色权限
│  ├─ observability/  ← logging/metrics/tracing
│  └─ ui/             ← 共享 UI 组件（布局、导航、表格壳）
│
├─ flows/             ← 业务流组件（将来每个可抽为独立包）
│  └─ npq/            ← 新品质量策划
│     ├─ projects/    ← 项目管理看板
│     ├─ tasks/       ← 任务流转
│     └─ stages/      ← 阶段模板与进度
│
└─ admin/             ← 后台管理（将来可抽为 @qe/admin）
   ├─ templates/      ← 阶段模板配置
   ├─ users/          ← 用户管理
   └─ dashboard/      ← 使用情况看板
```

### 依赖原则

`platform/` 和 `flows/` 单向依赖：flow 可以 import platform，不能反向。

## 十、核心工程原则

1. **代码干净可维护且灵活** — 最高优先级
2. 命名清晰 > 注释
3. 模块职责边界明确，改一个不波及全局
4. 新人/三个月后的自己能在一小时内理解核心结构
5. 数据模型一步到位，为未来集成预留字段
6. 所有核心能力优先本地实现，不强依赖外部平台
